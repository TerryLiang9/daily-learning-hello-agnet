# 第二章 智能体发展史 - 实践项目

本章节包含两个核心实践项目,帮助理解智能体的发展历程:

## 项目一:增强版ELIZA聊天机器人

基于符号主义时代的经典ELIZA程序,添加新规则和上下文记忆功能。

### 核心功能

- ✅ 基于模式匹配的对话系统
- ✅ 多规则优先级处理
- ✅ 代词转换机制
- ✅ 上下文记忆系统(记住用户信息)
- ✅ 对话历史记录
- ✅ 置信度评分

### 运行方法

```bash
# 基础版ELIZA
python eliza_basic.py

# 增强版(带记忆)
python eliza_advanced.py
```

## 📖 学习重点

### 1. 符号主义的核心思想
- 物理符号系统假说(PSSH)
- 知识库 + 推理机
- IF-THEN产生式规则
- 局限性:知识获取瓶颈、框架问题

### 2. ELIZA的工作原理
- 模式匹配与文本替换
- 关键词识别与优先级排序
- 分解模式(通配符捕获)
- 重组模式(模板填充)
- 代词转换

### 3. 规则系统的根本局限
- 缺乏语义理解
- 无上下文记忆(无状态)
- 规则扩展性问题(组合爆炸)

## 🎯 实践习题

### 习题1:扩展ELIZA规则

为`eliza_advanced.py`添加3-5条新规则,处理更多场景:
- 谈论工作(如"我在工作"、"我的工作")
- 谈论学习(如"我在学习"、"我想学")
- 谈论爱好(如"我喜欢运动"、"我的爱好")

### 习题2:对比ELIZA与ChatGPT

从以下维度对比增强版ELIZA与现代LLM(如ChatGPT):
1. **语义理解能力**:能否理解否定、反讽?
2. **上下文记忆**:能否进行连贯的多轮对话?
3. **知识范围**:是依赖预设规则还是拥有世界知识?
4. **扩展性**:添加新能力的难易程度?

### 习题3:分析组合爆炸问题

假设我们要覆盖10个话题(工作、学习、家庭、爱好等),每个话题有10种表达方式,每种表达有5种句式变种。如果要覆盖所有组合,需要多少条规则?
