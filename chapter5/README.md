# 第五章：基于低代码平台的智能体搭建

> 使用图形化平台快速构建智能体应用

**本章重点**：
- 了解低代码平台的价值和应用场景
- 学习Coze、Dify、n8n三大平台的使用
- 实践：构建"每日AI简报"助手

---

## 学习笔记

### 5.1 平台化构建的兴起

#### 为何需要低代码平台？

**核心价值**：

1. **降低技术门槛**：非技术人员也能参与构建
2. **提升开发效率**：快速验证想法，原型开发
3. **可视化与可观测性**：直观的调试体验
4. **标准化与最佳实践**：内置行业最佳实践

**适用场景**：
- 快速原型验证
- 业务人员自主开发
- 团队协作标准化
- 降低维护成本

### 5.2 低代码平台对比

| 平台 | 核心定位 | 特点 | 适用人群 |
|------|----------|------|----------|
| **Coze** | 零代码Agent构建 | 可视化界面、丰富插件、多平台发布 | 入门用户、产品经理、创作者 |
| **Dify** | 开源LLM应用平台 | 企业级、可扩展、RAG Pipeline | 开发者、企业团队 |
| **n8n** | 工作流自动化 | 连接SaaS服务、高度定制化 | 自动化开发者、企业 |

## 实践指南

### Coze平台实践

#### 功能模块对应

```
Coze组件 = 游戏元素

- 工作流 → 关卡通关路线图
- 对话流 → NPC对话通关
- 插件 → 角色技能卡
- 知识库 → 游戏百科全书
- 卡片 → 快捷道具栏
- 提示词 → 角色移动键
- 数据库 → 云存档
- 发布管理 → 关卡审核员
- 模型管理 → 游戏角色库/捏脸系统
- 效果评测 → 闯关评分系统
```

#### 创建"每日AI简报"助手

**目标**：
- 自动抓取36氪、虎嗅、IT之家、InfoQ、GitHub、arXiv等来源
- 整合AI领域头条新闻、学术论文、开源项目
- 生成结构化、专业化的简报

**实现步骤**：

1. **创建智能体**
   - 登录Coze平台
   - 点击"创建智能体"
   - 选择"单智能体自主规划模式"

2. **配置插件**
   - 搜索并添加新闻相关插件
   - 配置数据源（36氪、GitHub等）
   - 设置抓取频率和过滤条件

3. **设计工作流**
   - 创建信息收集工作流
   - 添加内容整合节点
   - 配置输出格式

4. **测试与发布**
   - 在测试环境中验证
   - 发布到微信/飞书等平台

### Dify平台实践

**核心功能**：
- Agent工作流编排
- RAG（检索增强生成）Pipeline
- 数据标注与模型微调
- API接口管理

**适用场景**：
- 企业级AI应用
- 需要私有化部署
- 高度定制化需求

### n8n平台实践

**核心优势**：
- 连接数百个SaaS服务
- 灵活的工作流自动化
- 强大的集成能力

**典型用例**：
- AI + 业务流程自动化
- 跨平台数据同步
- 智能通知系统

## 实践练习

### 练习1：Coze入门
1. 注册Coze账号
2. 创建一个简单的对话机器人
3. 添加至少2个插件
4. 发布到微信或飞书

### 练习2：Dify工作流

4. 测试检索效果

### 练习3：n8n自动化
1. 安装n8n
2. 创建一个简单的自动化流程
3. 集成LLM节点
4. 连接至少2个外部服务

## 学习总结

### 核心收获

1. **平台选择策略**：
   - 快速验证想法 → Coze
   - 企业级应用 → Dify
   - 复杂自动化 → n8n

2. **低代码vs代码**：
   - 低代码：快速、可视化、易维护
   - 代码：灵活、强大、深度定制
   - 结合使用，发挥各自优势

3. **未来趋势**：
   - 平台功能越来越强大
   - AI能力深度集成
   - 更低门槛，更高效能
